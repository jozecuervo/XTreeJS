# XTreeJS Roadmap

This roadmap defines product outcomes and release gates. Execution details belong in `TODO.md`.

## Now (0.1.x Stability)

Objective: make `0.1.x` reliable for daily terminal use.

Exit criteria:
- Resolve current known bugs listed in `TODO.md`.
- Keep test suite passing in CI on all merges to `main`.
- Ensure all delete/prune paths remain safe-by-default.

## Next (0.2.x UX Completion)

Objective: close high-value XTree parity gaps that improve navigation speed and operational confidence.

Exit criteria:
- Deliver autoview and split screen core flows.
- Add copy/move progress feedback for large operations.
- Ship jump-to-directory workflow with prompt history.

## Later (0.3.x Platform and Extensibility)

Objective: broaden platform support and customizability.

Exit criteria:
- Cross-platform support baseline (including Windows).
- Configurable keybindings and startup options.
- Persistent session state (last directory, sort/filter, tags).
